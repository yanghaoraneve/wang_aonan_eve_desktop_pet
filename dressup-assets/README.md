# Wang Aonan EVE Chibi Desktop Pet Assets

This folder contains the first usable resource pass for the dress-up desktop pet.

## Current Assets

### Animated outfits

- `animated-manifest.json`
- `animated/red_white_dress/spritesheet.webp`
- `animated/black_leather_headband/spritesheet.webp`
- `animated/bamboo_flute_brown/spritesheet.webp`

Each animated outfit is a complete 9-row desktop-pet atlas generated by the hatch-pet workflow. The atlas size is `1536x1872`, with `192x208` cells and the standard state row order: idle, running-right, running-left, waving, jumping, failed, waiting, running, review.

Each outfit folder also includes:

- `contact-sheet.png`: QA overview of all animation rows.
- `validation.json`: atlas validation report.

### Static skins

- `skins/01_red_white_dress.png`
- `skins/02_blonde_white_outfit.png`
- `skins/03_black_leather_headband.png`
- `skins/04_dotted_scarf_denim.png`
- `skins/05_bamboo_flute_brown.png`

Each skin is a full-body transparent PNG normalized to `768px` height. The runtime can display them at the recommended `320px` height and switch the `file` path listed in `manifest.json` to change outfits.

## Manifest

`manifest.json` describes:

- skin id and display name
- PNG file path
- original concept-sheet crop
- sprite size
- bottom-center anchor point
- recommended display height

Use the anchor as the pet's standing point when placing it on screen.

## Planned Layer Layout

The `layers/` directories are reserved for the next pass:

- `base/`: body silhouette and shared face
- `hair/`: red, blonde, black hair layers
- `outfits/`: separate clothes
- `accessories/`: earrings, necklaces, headband, microphone
- `expressions/`: idle, blink, happy, cool, singing

The `animations/` directories are reserved for frame sequences:

- `idle/`
- `click/`
- `drag/`

For now, the full-body static skins are ready for a simple outfit-switching prototype, and the animated outfit atlases are ready for full animation-based outfit switching.
